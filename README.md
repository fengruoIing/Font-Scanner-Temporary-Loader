# ASS/SSA 字体扫描与临时加载工具

> 字幕字体扫描 → 缺失匹配 → 临时挂载（GDI 会话级） → 一键卸载，零残留

Windows 上播放或编辑 ASS/SSA 字幕时，缺少对应的字体文件会导致显示乱码。传统做法是把字体安装到 `C:\Windows\Fonts`，但频繁装卸会污染系统字体库。

本工具提供一套 **纯用户态、免管理员权限** 的临时挂载方案，扫描字幕文件、匹配本地字体库、通过 `AddFontResourceExW` + `FR_PRIVATE` 加载到当前 GDI 会话，用完后一键卸载。

## 2. 快速开始

### 方式一：直接运行 exe（推荐）

从 GitHub 下载 [`dist/ASS-Fonts-Loader.exe`](dist/ASS-Fonts-Loader.exe)（约 8 MB），**双击运行**即可，无需安装任何环境。

### 方式二：运行源码

### 主程序（推荐）

```powershell
python ass_fonts_loader.py
```

**典型流程：**

| 步骤 | 操作 | 说明 |
|:--:|------|------|
| 1 | 选择**字幕文件夹** | 包含 `.ass` / `.ssa` 文件的目录 |
| 2 | 点击 **🔍 扫描字幕字体** | 解析所有字幕文件，列出用到的字体 |
| 3 | 选择**字体源文件夹** | 存放 `.ttf` / `.otf` / `.ttc` / `.fon` 的本地仓库 |
| 4 | 点击 **▶ 临时加载缺失字体** | 弹出确认框，确认后挂载到当前会话 |
| 5 | 打开播放器 / Aegisub | **重启目标软件**后字体生效 |
| 6 | 点击 **✖ 一键卸载所有临时字体** | 清理 GDI 资源和注册表项，系统恢复 |

> 切换字幕文件夹时，会自动询问是否需要重新扫描。

## 4. 核心功能

### 4.1 字幕字体解析

- 支持 `.ass`（Advanced SubStation Alpha）与 `.ssa` 格式
- 自动识别 `[V4+ Styles]`、`[V4 Styles]`、`[V3 Styles]`、`[V2 Styles]` 段落
- 通过 `Format` 行定位 `Fontname` 字段的列索引，**不依赖固定列顺序**
- 兼容 **UTF-8 with BOM / UTF-8 / GBK** 多种编码（按序回退）
- `get_fonts.py` 额外支持 `\fn` 内联标签的字体提取

### 4.2 字体匹配算法

在字体源目录中 **递归搜索**，针对每个缺失字体名按优先级打分：

| 优先级 | 策略 | 分值 |
|:--:|------|:--:|
| 1 | 去除变体后缀后基础名精确匹配 | +100 |
| 2 | Cleaned 字符串子串包含 | +80 |
| 3 | 基础名不区分大小写包含 | +60 |

- 自动过滤的变体后缀：`W3` `W5` `W7` `W9` `RD` `SB` `LT` `MD` `BK`
- 打分字段经 `_clean()` 处理：去除空白、`-`、`_`、`.`，转小写
- 最低接受分值 **≥ 80**（即至少子串级别匹配才通过）

### 4.3 加载确认对话框

点击 "临时加载缺失字体" 时，弹出确认框列出：
- 字体源文件夹路径
- 即将加载的字体数量
- **每条字体**的名称及匹配到的物理文件名
- 底部确认按钮

确认后执行以下操作：
1. 将字体路径写入 `HKCU\Software\Microsoft\Windows NT\CurrentVersion\Fonts`
2. 调用 `gdi32.AddFontResourceExW(font_file, FR_PRIVATE, 0)` 加载到 GDI
3. 如果 API 失败，自动回滚该字体的注册表项

### 4.4 一键卸载

1. `RemoveFontResourceExW` 撤销 GDI 资源占用
2. 批量删除 `HKCU\...\Fonts` 中的临时注册表项
3. 发送 `WM_FONTCHANGE` 广播消息通知系统刷新字体缓存

## 6. 临时加载原理

### 为什么不直接复制到 C:\Windows\Fonts？

直接写入系统字体目录需要 **管理员权限**，且字体永久残留，污染系统库。

### HKCU + FR_PRIVATE 方案

| 特性 | 说明 |
|------|------|
| **用户级隔离** (HKCU) | 仅当前用户可见，不影响其他用户 |
| **会话级有效** (FR_PRIVATE = 0x10) | 字体加载到 GDI 私有链表，仅当前会话期间有效 |
| **自动清理** | 注册表关机即失；GDI 资源随进程退出或手动卸载时释放 |

### 注册表写入格式

```
Key:   HKEY_CURRENT_USER\Software\Microsoft\Windows NT\CurrentVersion\Fonts
Value: "{FontDisplayName} (TrueType)"
Type:  REG_SZ
Data:  "C:\absolute\path\to\font.ttf"
```

### 相关 Windows API

```c
// 加载字体到进程私有链表（不注册到系统范围）
int AddFontResourceExW(LPCWSTR pFileName, DWORD fl, PDWORD pdv);
// fl = FR_PRIVATE (0x10) — 仅当前进程/会话可见

// 撤销加载
BOOL RemoveFontResourceExW(LPCWSTR pFileName);

// 通知所有顶层窗口刷新字体缓存
SendMessageW(HWND_BROADCAST, WM_FONTCHANGE, 0, 0);
```

### 线程模型

所有操作在主线程上 **同步阻塞** 执行，未使用 `threading` 或 `asyncio`。扫描过程中通过 `root.update()` 手动泵送事件循环以保持界面响应。`_scanning` 布尔锁防止重复进入扫描流程。

## 7. 故障排查

### Q1: 匹配不到字体文件？

- 确认字体源文件夹包含子目录（已支持递归搜索 `os.walk`）
- 检查文件名是否有过多特殊符号干扰匹配算法
- 可在报告区查看每个缺失字体的匹配状态及匹配得分

### Q2: AddFontResourceEx 返回 0（失败）？

常见 Windows 错误码：

| 错误码 | 含义 | 解决方案 |
|:--:|------|------|
| 5 | 拒绝访问 | 以管理员身份运行（罕见，通常无需） |
| 87 | 参数无效 | 字体文件损坏或非标准格式 |
| 2 | 找不到文件 | 路径含特殊字符，尝试移动文件到简单路径 |

### Q3: 加载成功但播放器里没变？

Aegisub、PotPlayer、MPC-HC 等软件缓存了字体列表，**重启目标软件** 即可生效。加载动作在 GDI 层面有效，但应用进程需要重新枚举字体。

### Q4: 卸载后系统字体未刷新？

程序已自动发送 `WM_FONTCHANGE` 广播消息。极少数情况下需手动重启资源管理器（`explorer.exe`）或注销重新登录。

### Q5: 程序崩溃后注册表残留？

HKCU 下的临时字体注册表项在程序崩溃时不会自动清理，但它们 **无害**，重启 Windows 即自动清除（HKCU 注册表在用户注销时释放）。
